assets/img/projects/mapit.jpg MapIt Web application to guide and share personal studies through interactive roadmaps ASP.NET HTML CSS JavaScript MySQL Cloudflare
Developed as a high school group project, one of the main problems we faced constantly being students ourselves was organizing study materials, we found the traditional methods to be often complicated to maintain and backtrack. We wanted to have a flexible and easy way for anyone to create and share their studies, and that's when we found the format we would eventually settle with, one that was already popular within professional spaces but mostly unseen in educational settings, Roadmaps. Mapit was structured as being simply a platform for roadmaps, this means granting users all the basic functions such as creating, sharing, deleting, editing and even giving feedback on roadmaps. The main layout of the application was designed akin to other similar social medias, each roadmap has their own tags and are part of a category, they have features to assist in providing feedback in ratings and comments, users have their own profile that displays roadmaps, comments and ratings.
  • Technical leadership: Acted as the primary architect, aligning the team's knowledge with the project's technical needs
  • Core Logic Development: Developed a custom Javascript library for interactive roadmap manipulation using a GUI
  • Cloud storage: Integrated Cloudflare R2 for scalable file storage and dedicated Workers for custom logic
  • System architecture: Engineered middleware and utility layers in the ASP.NET backend and Javascript frontend for project-wide consistency
  • Database Engineering: Wrote high-performance MySQL stored procedures and triggers, prioritizing performance and optimization
assets/img/projects/farmafacil.png Farma Fácil Node.js API for consulting medicine prices across multiple online pharmacies Node.js SQLite Express JWT
The concept and development took under a week, we were participating in a coding competition and needed to find what would be the simplest thing that could help the most people, and thus Farma Fácil was made, an API for quickly consulting medicine prices across multiple pharmacies, smoothing out a normally tedious task. As per the competition's rules, we would only develop a Node API and the frontend would come at a later stage. The API was designed with ease of use in mind, using simple web scraping for obtaining and filtering prices and JWT authentication for admin management.
  • Web scraping: Developed the API's web scraper using NPM libraries such as Cheerio
  • Endpoint development: Wrote all endpoints of the project using Express and JWT middleware for admin routes
  • Database architecture: Engineered the project's SQLite database for keeping track of admins and pharmacies
assets/img/projects/aloprefeitura.png Alô Prefeitura Mobile app for reporting issues to the city government Node.js Express MySQL Cloudflare Ionic React
The application was made as part of a school group project where we would have to pick a theme and then make any sort of project based on that, we picked "IOT and smart cities" and quickly recognized an issue that even our school was dealing with at the time, the difficult and unintuitive process of reporting problems to city council, that's when we got our fist concept of a mobile app that could greatly facilitate that activity. The app follows a typical social media structure, having a feed containing the latest posts (in this case, complaints) which can be liked or disliked to gauge public priority, every post must contain a location and picture related to the problem, city hall workers have admin accounts in which they can check the complaints and mark them as sorted out.
  • Node Backend: Developed the application's backend API using Node JS and Express
  • Cloud storage: Integrated Cloudflare R2 for storing post images and profile pictures using Workers
  • Database development: Engineered the project's MySQL database including the stored procedures
  • Frontend architecture Designed the React frontend's structure, wrote middleware and utility functions
